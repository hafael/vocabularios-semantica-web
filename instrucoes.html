Manual do Ambiente Semântico Web

Organização e Desenvolvimento de Vocabulários Controlados e Ontologias

Desenvolvido por: Rafael Villa Verde (rafaelverde@id.uff.br)

Este ambiente foi projetado para pesquisadores de Ciência da Informação e Biblioteconomia que desejam trabalhar com a Web Semântica. Ele contém todas as ferramentas necessárias para criar, gerenciar e publicar dados interconectados (Linked Data).

1. Baixando os Arquivos do Projeto

O primeiro passo é obter os arquivos de configuração que unem todos os softwares abaixo.

  1. Acesse o endereço do projeto no GitHub (Ex: https://github.com/hafael/vocabularios-semantica-web).
  2. Clique no botão verde escrito "<> Code" (ou Código).
  3. No menu que abrir, clique em "Download ZIP".
  4. Após baixar, extraia o arquivo ZIP em uma pasta fácil de encontrar no seu computador (como C:\TrabalhoSemantico).
Ir para o GitHub

2. Instalando o "Motor" (Docker Desktop)

O Docker funciona como o coração do nosso sistema. Ele permite rodar servidores complexos (como o Fuseki e o WebProtégé) sem que você precise instalar e configurar cada um deles manualmente no seu Windows.

Para instalar:

  1. Baixe o instalador no site oficial: Docker Desktop.
  2. Execute o instalador e aceite todas as configurações padrão.
  3. Atenção (Usuários Windows): O Docker exige um componente chamado WSL 2. Se o instalador solicitar ou der erro, baixe e instale manualmente aqui:
    👉 Link para Atualização do WSL 2 (Kernel da Microsoft)
  4. Ao final, o Windows pode pedir para reiniciar. Isso é normal!

3. Ferramentas de Edição e Catálogo

Para complementar o servidor, instalaremos duas aplicações locais no seu desktop:

Visual Studio Code (VS Code)

Nosso editor de "baixo nível". Serve para visualizarmos e editarmos arquivos de dados RDF (como Turtle e JSON-LD) e o arquivo de configuração .env com facilidade. Baixar VS Code

MarcEdit 7

Ferramenta essencial para Bibliotecários. Ela permite manipular registros bibliográficos no formato MARC21 e convertê-los para formatos semânticos, conectando a catalogação tradicional à Web de Dados. Baixar MarcEdit 7

4. Iniciando os Servidores (O Momento Crucial)

Este é o passo onde a "mágica" acontece: os arquivos que você baixou vão instruir o Docker a montar todo o seu laboratório de dados. Escolha o seu sistema operacional abaixo:

💻 Se você usa Windows

  1. Abra a pasta do projeto (onde você extraiu o ZIP).
  2. Ação: Localize e execute o arquivo abaixo:
    start.bat
  3. Como? Basta dar um clique duplo sobre ele.
  4. ⚠️ Aviso de Segurança: Se o Windows exibir uma tela azul ("O Windows protegeu o seu computador"), clique no link "Mais informações" e depois no botão "Executar assim mesmo".

🍎 Se você usa macOS

  1. Abra o Terminal (Aplicativos > Utilitários).
  2. Digite cd (com espaço no fim) e arraste a pasta do projeto para o terminal. Pressione Enter.
  3. Ação 1 (Permissão): Copie, cole e dê Enter: 
    chmod +x start.sh
  4. Ação 2 (Iniciar): Digite o comando principal:
    ./start.sh
  5. Mantenha esta janela aberta até terminar seu trabalho.

O que esperar agora? na primeira vez que você rodar, o Docker vai baixar cerca de 500MB de dados da internet. Isso pode levar alguns minutos dependendo da sua conexão. Você verá várias linhas escritas "Pulling", "Downloading" ou "Extracting".

Quando termina? Quando o Terminal parar de se mexer e exibir a mensagem: Ambiente pronto!

6. Concluindo a Configuração Inicial (Setup)

Conforme a documentação oficial, após iniciar o servidor pela primeira vez, é necessário configurar a conta de administrador e as permissões globais.

  1. Acessar o Terminal: Com os servidores ligados, abra o Terminal (macOS) ou PowerShell/CMD (Windows).
  2. Criar Conta de Administrador: Execute o comando abaixo para iniciar o assistente de criação de conta no banco de dados do WebProtégé:
    docker exec -it webprotege java -jar /webprotege-cli.jar create-admin-account
  3. Siga os Prompts: O sistema solicitará um Nome de Usuário, Endereço de E-mail e Senha. Preencha e confirme.
  4. Configurações Globais: Faça login no WebProtégé e acesse o menu de configurações da aplicação em:
    http://localhost:5000/#application/settings
  5. Permissões: Preencha os campos obrigatórios e marque todos os itens na seção "Permission" (Permissões) para garantir acesso administrativo total.

Nota: Esta etapa é crucial para que você possa criar novos projetos e gerenciar usuários no seu laboratório semântico.

🔄 Reiniciando do Zero (Reset)

Se você precisar apagar TODOS os seus dados, ontologias e configurações para começar do zero (como se tivesse acabado de baixar o projeto):

Windows: Localize e execute o arquivo abaixo:
reset.bat
macOS: No Terminal, dentro da pasta do projeto, digite:
./reset.sh

⚠️ Cuidado: Esta ação é permanente e não pode ser desfeita!

❓ Perguntas e Respostas Frequentes (FAQ)

1. O meu painel de monitoramento está "Desconectado", o que eu faço?
Primeiro, verifique se o Docker Desktop está aberto e com a "baleia" verde. Se estiver tudo certo, espere mais 2 minutos; o WebProtégé e o Fuseki podem demorar um pouco para carregar totalmente na primeira vez.

2. O arquivo start.bat abre e fecha muito rápido, sem fazer nada.
Isso geralmente acontece se o Docker não estiver rodando. Certifique-se de abrir o Docker Desktop antes de clicar no start.bat.

3. Recebi um erro de "Port already in use" (Porta em uso).
Isso significa que outro programa no seu computador já está usando a porta 3030 ou 5000. Você pode resolver isso fechando o outro programa ou mudando o número da porta no arquivo .env (ex: mude 3030 para 3031) e rodando o start.bat novamente.

4. Minha senha admin/admin não funciona no WebProtégé.
Isso ocorre se a automação falhar na primeira subida. Siga os passos de Configuração Inicial (Setup) descritos no Passo 6 deste manual para criar sua conta de administrador manualmente. Se ainda assim falhar, use o script de Reset e tente novamente.

5. O Windows deu um erro de "WSL 2 installation is incomplete".
Baixe e instale o pacote de atualização oficial da Microsoft neste link: Atualização do Kernel do WSL 2. Após instalar, reinicie o computador.
