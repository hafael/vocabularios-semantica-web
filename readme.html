Sobre o Ambiente de Desenvolvimento

Universidade Federal Fluminense (UFF)
Curso de Biblioteconomia e Documentação
Disciplina: Vocabulários e Semântica na Web (GCI00189)
Responsável: Profª. Drª. Rosana Portugal Tavares de Moraes

Este projeto foi concebido para automatizar e simplificar a configuração de um laboratório de Web Semântica. Em vez de instalações complexas e manuais, o ambiente utiliza Docker para orquestrar serviços de triplestore e editores de ontologias de forma isolada e persistente.

🛠️ Tecnologias Utilizadas

  • Docker & Docker Compose
  • Apache Jena Fuseki (Triplestore)
  • WebProtégé (Editor de Ontologias)
  • MongoDB (Persistência)
  • Bash/Batch (Automação)
  • SPARQL / RDF / Turtle

📂 Estrutura de Diretórios

O ambiente foi desenhado para manter os dados separados do sistema, permitindo backups fáceis:

  • fuseki-data/: Onde residem os bancos de dados TDB2 e arquivos de configuração de usuários.
  • webprotege-data/: Pasta para os arquivos de projetos de ontologias criados no WebProtégé.
  • mongo-data/: Banco de dados interno do WebProtégé para gerenciar usuários e metadados.
  • scripts/: Automação para setup e permissões de sistema.

🎓 Objetivos Pedagógicos

O objetivo principal é permitir que o aluno foque na modelagem conceitual e estrutural de dados semânticos, sem se preocupar com as barreiras técnicas de instalação de servidores Java ou gerenciamento de bancos de dados.

🤖 Co-desenvolvimento com IA

Este ambiente de desenvolvimento foi projetado e implementado em parceria com o assistente de inteligência artificial Gemini CLI.

A arquitetura dos containers, scripts de automação e toda a documentação foram gerados e refinados utilizando os modelos Gemini 2.0 Flash (para execução rápida e precisa) e Gemini 1.5 Pro (para planejamento estrutural e raciocínio complexo).
